## Regla principal

Para obtener una nota alta no basta con ejecutar el juego. Debes poder:

1. Explicar qué problema resuelve.
2. Describir la lógica sin leer el código.
3. Justificar por qué usaste cada ciclo.
4. Hacer una traza manual con un ejemplo.
5. Modificar una parte pequeña, compilar y demostrarla en 5 minutos.

## Método para cada sesión

Usa este ciclo:

1. Lee una sección corta.
2. Cierra el documento.
3. Explícala con tus propias palabras.
4. Abre `codigo.cpp` y señala dónde aparece esa lógica.
5. Ejecuta un ejemplo.
6. Anota qué parte todavía no puedes explicar.

No estudies únicamente leyendo. La evaluación incluye preguntas y una modificación presencial.

## Semáforo de preparación

Marca cada punto:

| Estado | Significado |
| :--- | :--- |
| Rojo | No puedo explicarlo todavía. |
| Amarillo | Lo entiendo, pero necesito mirar el código. |
| Verde | Puedo explicarlo y modificarlo sin ayuda. |

Debes llegar en verde en estos temas:

- Flujo de `main`.
- Uso de prototipos.
- Diferencia entre definición y llamada de una función.
- Uso de `while`, `do while` y `for`.
- Diferencia entre `=` y `==`.
- Uso de `&&` y `||`.
- Extracción de dígitos con `% 10`.
- Eliminación de dígitos con `/ 10`.
- Cálculo de pistas.
- Validación de intentos.
- Puntaje.
- Compilación después de una modificación.
