# Análisis del enfoque del profesor

Las lecciones disponibles muestran cuatro prioridades.

## 1. Entender antes de programar

Debes poder explicar:

```text
El jugador ingresa un código.
El programa valida el código.
Si falla, muestra pistas.
El proceso se repite mientras queden intentos.
```

## 2. Elegir bien los ciclos

| Ciclo | Uso en el proyecto |
| :--- | :--- |
| `do while` | Mostrar menú al menos una vez. |
| `while` | Repetir validaciones y turnos. |
| `for` | Comparar exactamente cinco posiciones. |

## 3. Dominar dígitos

```cpp
digito = numero % 10;
numero = numero / 10;
```

Este patrón permite recorrer números sin arrays.

## 4. Adaptar condiciones

La modificación presencial probablemente cambiará una condición pequeña:

- Límite de intentos.
- Tipo de pista.
- Código secreto.
- Regla de validación.

No memorices líneas. Comprende el objetivo de cada función.
